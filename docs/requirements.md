# 미국 투자 포트폴리오 자동 관리 툴 요구사항

## 1. 목표

한국투자증권 API와 텔레그램을 연동하여 미국 대형주 터틀 추세추종 전략을 자동으로 실행하는 포트폴리오 관리 툴을 만든다.

이 툴은 매일 장 시작 전 필요한 거래를 판단하고, 사용자 승인 후 실제 주문을 실행하며, 장 종료 후 실행 결과를 보고한다. 미완료 거래가 있으면 다음 거래일에 다시 승인 절차부터 반복한다.

## 2. 핵심 사용자 흐름

1. 시스템이 정해진 주기에 따라 포트폴리오, 시세, 전략 데이터를 갱신한다.
2. 미국 장 시작 전에 당일 필요한 매수, 매도, 추가 진입, 손절, 리밸런싱 주문 후보를 계산한다.
3. 주문 후보와 예상 포트폴리오 변화를 텔레그램으로 사용자에게 보낸다.
4. 사용자가 텔레그램에서 승인하면 시스템이 한국투자증권 API로 실제 주문을 실행한다.
5. 미국 장 종료 후 체결 내역, 미체결 내역, 포트폴리오 상태, 다음 조치 필요 여부를 텔레그램으로 보고한다.
6. 미체결 또는 미완료 주문이 있으면 다음 거래일 장 시작 전 승인 절차부터 다시 진행한다.

## 3. 외부 연동

### 3.1 한국투자증권 API

- 미국 주식 계좌 잔고 조회
- 미국 주식 현재가 및 과거 일봉 데이터 조회
- 주문 가능 금액 조회
- 미국 주식 매수, 매도 주문
- 주문 체결 및 미체결 내역 조회
- API 인증 토큰 발급, 갱신, 만료 처리
- 실전/모의투자 환경 분리

### 3.2 텔레그램

- 당일 주문 후보 승인 요청 메시지 발송
- 승인, 거절, 보류 명령 수신
- 장 종료 후 결과 보고 메시지 발송
- 오류, API 장애, 주문 실패 등 예외 상황 알림

## 4. 투자 전략

### 4.1 전략명

미국 대형주 터틀 추세추종 전략 v1.0

### 4.2 유니버스

- 대상: 미국 증시 시가총액 상위 10개 개별 종목
- 갱신 주기: 매년 1월 1일
- 유니버스 산정 시점과 데이터 출처를 기록한다.
- 유니버스 변경 이력은 추적 가능해야 한다.

### 4.3 진입 및 청산 규칙

- System 1 진입: 최근 20일 최고가 상향 돌파 시 1 Unit 매수
- 청산: 최근 10일 최저가 하향 이탈 시 해당 종목 전량 매도
- 손절: 마지막 진입가 대비 `-2 ATR` 도달 시 해당 종목 전량 매도
- ATR 기준: 20일 평균 ATR

### 4.4 자금 관리

- 회당 리스크: 총 자산의 1%
- 종목당 최대 유닛 수: 4 Unit
- 추가 진입: 직전 진입가 대비 `+0.5 ATR` 상승할 때마다 1 Unit 추가
- 포지션 사이징:

```text
Unit Size = (Total Equity * 0.01) / (N * Price_Point_Value)
N = 20일 평균 ATR
```

- 미국 주식은 주문 단위, 소수점 거래 가능 여부, 최소 주문 금액을 반영한다.
- 환율, 달러 예수금, 원화 예수금은 계좌 기준으로 명확히 구분한다.

### 4.5 현금 관리

- 매수 대기 자금은 파킹형 ETF로 운용한다.
- 후보 ETF: SGOV, KOFR 등
- 실제 운용 대상 ETF, 우선순위, 매수/매도 규칙은 별도 설정으로 관리한다.
- 주식 매수 필요 시 파킹형 ETF 매도 후 현금 확보 절차를 고려한다.

## 5. 주문 승인 정책

- 실제 주문 전 반드시 사용자 승인이 필요하다.
- 승인 요청은 미국 정규장 시작 전에 텔레그램으로 발송한다.
- 승인 메시지에는 다음 정보를 포함한다.
  - 주문 종목
  - 매수/매도 구분
  - 수량
  - 주문 유형
  - 기준 가격
  - 전략상 근거
  - 예상 리스크
  - 주문 후 예상 포트폴리오 비중
- 사용자가 승인하지 않으면 실제 주문을 실행하지 않는다.
- 거절된 주문은 기록하고, 다음 판단 시점에 전략 조건을 다시 평가한다.

## 6. 장 종료 보고

장 종료 후 사용자에게 다음 내용을 보고한다.

- 당일 승인된 주문 목록
- 체결 주문
- 미체결 주문
- 실패 주문과 실패 사유
- 현재 보유 종목 및 비중
- 현금 및 파킹형 ETF 상태
- 전략상 다음 거래일에 재시도해야 할 항목

## 7. 반복 처리

- 주문이 완료되지 않았거나 일부만 체결된 경우 다음 거래일에 다시 승인 절차부터 반복한다.
- 반복 대상은 단순 재주문이 아니라, 다음 거래일 최신 가격과 전략 조건을 반영하여 재계산한다.
- 반복 이력은 감사 가능하도록 저장한다.

## 8. 데이터 및 상태 관리

시스템은 최소한 다음 데이터를 저장한다.

- API 인증 상태
- 계좌 잔고 스냅샷
- 유니버스 이력
- 일봉 가격 데이터
- ATR, 20일 최고가, 10일 최저가 등 전략 지표
- 포지션별 유닛 수
- 마지막 진입가
- 주문 후보
- 사용자 승인/거절 이력
- 주문 요청, 체결, 미체결, 실패 이력
- 장 종료 보고 이력

## 9. 스케줄링

- 유니버스 갱신: 매년 1월 1일
- 전략 판단: 미국 정규장 시작 전
- 승인 요청: 미국 정규장 시작 전
- 주문 실행: 사용자 승인 후, 정규장 주문 가능 시간
- 결과 보고: 미국 정규장 종료 후
- 미완료 주문 재평가: 다음 거래일 장 시작 전

미국 휴장일, 조기 폐장일, 서머타임을 반영해야 한다.

## 10. 안전장치

- 실전 주문과 모의 주문 환경을 명확히 분리한다.
- 기본 실행 모드는 모의투자 또는 드라이런으로 시작한다.
- 동일 주문 중복 실행 방지 장치가 필요하다.
- 당일 승인 범위를 초과하는 주문은 실행하지 않는다.
- API 오류, 네트워크 장애, 주문 실패 시 사용자에게 즉시 알린다.
- 주문 실행 전 계좌 상태를 재조회하여 수량과 가능 금액을 검증한다.
- 모든 주문 결정과 실행 결과는 로그로 남긴다.

## 11. 설정

설정 파일 또는 환경 변수로 다음 항목을 관리한다.

- 한국투자증권 API 앱 키, 앱 시크릿
- 계좌번호
- 실전/모의투자 모드
- 텔레그램 봇 토큰
- 텔레그램 사용자 또는 채팅 ID
- 타임존
- 주문 유형 기본값
- 파킹형 ETF 목록
- 최대 일일 주문 금액 또는 비중 제한
- 승인 응답 만료 시간

민감 정보는 저장소에 커밋하지 않는다.

## 12. 비기능 요구사항

- 전략 계산은 재현 가능해야 한다.
- 주문 실행은 멱등성을 가져야 한다.
- 로그와 저장 데이터만으로 특정 주문의 판단 근거를 추적할 수 있어야 한다.
- 핵심 전략 로직은 단위 테스트로 검증한다.
- API 연동부는 모의 객체 또는 샌드박스 환경으로 테스트 가능해야 한다.
- 장애 발생 시 전체 시스템이 조용히 실패하지 않고 사용자에게 보고해야 한다.

## 13. 미정 사항

- 한국투자증권 해외주식 주문에서 사용할 주문 유형
- 정규장, 프리마켓, 애프터마켓 중 주문 허용 시간
- 미국 시가총액 상위 10개 종목 데이터 출처
- 파킹형 ETF의 최종 선택과 운용 규칙
- 소수점 매매 사용 여부
- 환전 자동화 여부
- 배당금과 세금 반영 방식
- 배포 환경
