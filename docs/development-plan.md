# 개발 계획

## 방향

먼저 실제 주문을 막은 상태에서 전략 계산, 승인 흐름, 주문 시뮬레이션을 완성한다. 이후 한국투자증권 모의투자 환경에서 주문 실행을 검증하고, 마지막 단계에서 실전 주문을 제한적으로 활성화한다.

## 마일스톤 1. 프로젝트 기반 구축

### 목표

개발을 시작할 수 있는 기본 구조, 설정, 실행 방식을 만든다.

### 작업

- 애플리케이션 런타임과 프레임워크 결정
- 설정 파일 및 환경 변수 구조 설계
- 민감 정보 관리 방식 정의
- 로깅 구조 추가
- 기본 CLI 또는 스케줄러 진입점 생성
- 테스트 프레임워크 설정

### 산출물

- 실행 가능한 최소 프로젝트
- 예시 설정 파일
- 기본 로그 출력
- 초기 테스트 실행 환경

## 마일스톤 2. 전략 엔진 구현

### 목표

터틀 추세추종 전략 v1.0을 외부 주문 없이 순수 계산 로직으로 구현한다.

### 작업

- 일봉 데이터 모델 정의
- ATR 20일 계산
- 최근 20일 최고가 돌파 판단
- 최근 10일 최저가 이탈 판단
- `-2 ATR` 손절 판단
- `+0.5 ATR` 추가 진입 판단
- 종목당 최대 4 Unit 제한
- 총 자산 1% 리스크 기반 Unit Size 계산
- 전략 판단 결과를 주문 후보로 변환

### 산출물

- 전략 계산 모듈
- 주요 전략 규칙 단위 테스트
- 샘플 가격 데이터 기반 백테스트성 검증

## 마일스톤 3. 상태 저장소 구현

### 목표

포지션, 유닛, 승인, 주문, 보고 이력을 안정적으로 저장한다.

### 작업

- 데이터베이스 또는 파일 기반 저장소 결정
- 계좌 스냅샷 저장
- 유니버스 이력 저장
- 포지션별 유닛 수와 마지막 진입가 저장
- 주문 후보와 승인 이력 저장
- 체결, 미체결, 실패 이력 저장
- 멱등성 키 설계

### 산출물

- 상태 저장소
- 마이그레이션 또는 스키마 문서
- 주문 중복 방지 테스트

## 마일스톤 4. 한국투자증권 API 연동

### 목표

계좌, 시세, 주문 관련 API를 안전하게 호출할 수 있게 한다.

### 작업

- 인증 토큰 발급 및 갱신
- 해외주식 잔고 조회
- 주문 가능 금액 조회
- 미국 주식 일봉 조회
- 현재가 조회
- 모의투자 주문 API 연동
- 주문 체결 및 미체결 조회
- API 오류 처리와 재시도 정책 구현

### 산출물

- 한국투자증권 API 클라이언트
- 모의투자 계좌 조회 검증
- 모의 주문 실행 검증
- API 실패 케이스 테스트

## 마일스톤 5. 텔레그램 승인 흐름

### 목표

실제 주문 전 사용자 승인 절차를 완성한다.

### 작업

- 텔레그램 봇 메시지 발송
- 승인 요청 메시지 포맷 구현
- 승인, 거절, 보류 응답 수신
- 승인 만료 시간 처리
- 승인된 주문만 실행 대상으로 표시
- 승인 이력 저장

### 산출물

- 텔레그램 승인 봇
- 승인/거절/만료 흐름 테스트
- 주문 후보 메시지 템플릿

## 마일스톤 6. 주문 오케스트레이션

### 목표

전략 판단, 승인, 주문 실행, 결과 조회를 하나의 일일 운영 흐름으로 연결한다.

### 작업

- 장 시작 전 전략 판단 잡 구현
- 승인된 주문 실행 잡 구현
- 주문 직전 계좌 재검증
- 주문 수량 재계산 또는 수량 검증
- 중복 주문 방지
- 장 종료 후 체결 결과 조회
- 미완료 주문의 다음 거래일 재평가 플래그 저장

### 산출물

- 일일 자동 운영 파이프라인
- 드라이런 실행 결과
- 모의투자 일일 사이클 검증

## 마일스톤 7. 장 종료 보고

### 목표

사용자가 하루 운용 결과와 다음 조치를 한눈에 이해할 수 있게 한다.

### 작업

- 체결 주문 요약
- 미체결 주문 요약
- 실패 주문 및 원인 요약
- 현재 포트폴리오 비중 계산
- 현금 및 파킹형 ETF 상태 계산
- 다음 거래일 재시도 대상 표시
- 텔레그램 보고 메시지 발송

### 산출물

- 장 종료 보고서 템플릿
- 텔레그램 결과 보고
- 보고 이력 저장

## 마일스톤 8. 유니버스 및 현금 관리

### 목표

연간 유니버스 갱신과 파킹형 ETF 운용 규칙을 자동화한다.

### 작업

- 미국 시가총액 상위 10개 데이터 소스 결정
- 매년 1월 1일 유니버스 갱신 잡 구현
- 기존 보유 종목이 유니버스에서 제외될 때 처리 정책 구현
- SGOV, KOFR 등 파킹형 ETF 설정화
- 주식 매수 필요 시 현금 확보 로직 설계
- 파킹형 ETF 매수/매도 승인 흐름 포함

### 산출물

- 유니버스 갱신 기능
- 파킹형 ETF 운용 기능
- 유니버스 변경 이력 보고

### 목표

미국 시장 일정과 장애 상황을 반영해 운영 리스크를 줄인다.

- 네트워크 실패 재시도
- 실행 중단 후 재시작 복구

### 산출물

- 시장 캘린더 연동
- 운영 장애 알림
- 재시작 복구 테스트

## 마일스톤 10. 실전 전환 준비

### 목표

모의투자에서 검증된 흐름을 제한적으로 실전 주문에 적용할 준비를 마친다.

### 작업

- 실전 주문 모드 활성화 스위치 구현
- 일일 최대 주문 금액 또는 비중 제한
- 주문 전 최종 확인 메시지 강화
- 실전 주문 전 체크리스트 작성
- 소액 실전 주문 리허설
- 운영 로그 점검

### 산출물

- 실전 전환 체크리스트
- 제한적 실전 주문 기능
- 운영 매뉴얼 초안

## 우선순위

1. 전략 엔진과 상태 저장소를 먼저 구현한다.
2. 한국투자증권 API는 조회 기능부터 붙이고, 주문은 모의투자로 제한한다.
3. 텔레그램 승인은 실제 주문보다 먼저 완성한다.
4. 모든 주문 실행은 드라이런, 모의투자, 실전 순서로 단계적으로 연다.
5. 실전 전환 전에는 최소 2주 이상의 모의 운영 로그를 확인한다.

## 초기 기술 선택 후보

- Backend: Python 또는 Node.js
- Scheduler: APScheduler, cron, GitHub Actions, 서버 상주 프로세스 중 선택
- Storage: SQLite 우선 검토
- Notification: Telegram Bot API
- Broker: 한국투자증권 Open API
- Testing: 전략 로직 단위 테스트 + API 클라이언트 모의 테스트
